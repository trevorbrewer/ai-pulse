simonw/browser-compat-dbSimon Willison

Simon Willison published a new GitHub repository called browser-compat-db, which converts Mozilla's mdn/browser-compat-data repository into a queryable SQLite database. The conversion script was generated using Claude Code for web running Opus 4.8, and the resulting database weighs approximately 66 MB. The project was inspired by Mozilla's new MDN MCP service and uses the sqlite-utils library to make browser compatibility data more accessible for programmatic querying.

A defining design principle of the tool is that it should remain usable even if the software itself ceases to exist, meaning secrets are stored in a format recoverable without the application. OpenAI Codex bombards SSDs with needless write operationsHacker News

The Register reported that OpenAI Codex generates excessive and unnecessary write operations to SSDs during operation, a behavior that accelerates drive wear and has been costing users millions of dollars in infrastructure expenses. The issue points to a gap between how AI coding tools are optimized for output quality versus how they interact with underlying storage hardware. The problem is notable because SSD write endurance is finite, and workloads that amplify writes can significantly shorten drive lifespans in production environments.
