Open-Source Tools & Developer Projects

Show HN: FixBugs – Reproduce production bugs and verify fixesHacker News

FixBugs is an AI agent that ingests contextual data surrounding production bugs, reproduces them in a sandbox environment, and generates verified fixes. The tool is notable because it closes the loop between bug detection and confirmed resolution rather than merely suggesting patches. It is available both as a self-hosted VSCode extension and as a GitHub app, giving teams flexibility in how they integrate it into existing workflows.

Show HN: Sx 2.0 – Share AI skills with your team through a Dropbox folderHacker News

Sx 2.0 is a tool that allows teams to share AI skills across clients by storing them in a Dropbox folder, removing the requirement for git-based storage that characterized its first version. The update was driven by user feedback revealing that non-technical users were increasingly adopting AI skills but lacked a practical sharing mechanism. The new approach supports sharing at personal, team, and organizational levels without requiring command-line familiarity.

Cdbx.ai – AI-powered browser IDE to describe, build, and publish appsHacker News

Cdbx.ai is a browser-based IDE that lets users describe an application in natural language, generate it with AI assistance, and publish it directly from the same interface. The tool targets users who want to move from idea to deployed app without switching between separate editing, building, and hosting environments. Running entirely in the browser lowers the setup barrier compared with locally installed development tools.

DOOMQLSimon Willison

Developer Peter Gostev built DOOMQL, a Doom-like game in which SQLite serves as the game engine rather than just a data store, handling movement, collision detection, enemy logic, combat, and pixel rendering entirely through SQL queries. The project was built using GPT-5.6 Sol and is implemented as a Python terminal application. Safety Research

MIT's New Method Flags AI Models Trained on CASM Without Generating ItHacker News

Researchers at MIT have developed a technique to detect whether an AI model was trained on child sexual abuse material (CSAM) without requiring the model to generate any such content during the evaluation process. This is significant because prior detection approaches risked producing harmful outputs as a byproduct of probing model behavior. The method addresses a critical gap in AI model auditing, particularly as regulators and platforms increase scrutiny of training data provenance.
